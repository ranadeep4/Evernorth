JavaScript Async/Await Pattern

Why It Matters

  • All Playwright actions return Promises
  • Always await async operations
  • Cleaner than callback hell
  • Error handling with try/catch
// Async/Await pattern
async function fetchUserData() {
    try {
        const response = await fetch('/api/user');
        return await response.json();
    } catch (error) {
        console.error(error);
    }
}

// Parallel execution
await Promise.all([
    page.waitForURL(/dashboard/),
    page.click('button')
]);

Playwright Framework

Why Playwright?

  • Auto-wait - No manual waits needed
  • Multi-browser - Chrome, Firefox, Safari
  • Parallel - Fast test execution
  • Tracing - Debug failed tests easily
  • API Testing - Built-in request context
import { test, expect } from '@playwright/test';

test('Login Test', async ({ page }) => {
    await page.goto('/login');
    
    await page.getByPlaceholder('Username')
        .fill('Admin');
    await page.getByPlaceholder('Password')
        .fill('admin123');
    await page.getByRole('button', { name: 'Login' })
        .click();
    
    await expect(page.getByRole('heading'))
        .toBeVisible();
});

Cypress Framework

Cypress Strengths

  • Command Queue - Automatic retries
  • Time Travel - See every command
  • Real-time Reload - Fast feedback
  • Network Stubbing - cy.intercept power
  • Fixtures - Easy test data loading
describe('Login Tests', () => {
    it('should login successfully', () => {
        cy.visit('/login');
        cy.get('[name="username"]').type('Admin');
        cy.get('[name="password"]').type('admin123');
        cy.get('button[type="submit"]').click();
        
        cy.url().should('include', '/dashboard');
    });
    
    it('should intercept API', () => {
        cy.intercept('GET', '/api/users/**')
            .as('getUsers');
        cy.visit('/users');
        cy.wait('@getUsers')
            .its('response.statusCode')
            .should('equal', 200);
    });
});

Page Object Model - Our OrangeHRM Example

POM Benefits

  • Encapsulation - Locators in one place
  • Reusability - Share across tests
  • Maintainability - Change once, update all
  • Readability - Tests read like user stories

From: src/pages/loginpage.ts

// Page Object Model
export class LoginPage {
    #userName: Locator;
    #password: Locator;
    #loginButton: Locator;

    constructor(private page: Page) {
        this.#userName = page.getByPlaceholder('Username');
        this.#password = page.getByPlaceholder('Password');
        this.#loginButton = page.getByRole('button', 
            { name: 'Login' });
    }
    
    async login(user: string, pass: string) {
        await this.#userName.fill(user);
        await this.#password.fill(pass);
        await this.#loginButton.click();
    }
}

Custom Fixtures - Our Base Test Pattern

Why Fixtures?

  • Dependency Injection - POMs auto-created
  • Clean Tests - No setup boilerplate
  • Parallel Safe - Each test isolated
  • Lazy Loading - Only what you need

From: src/basetest.ts

// Custom fixture setup
import { test as base } from '@playwright/test';
import { LoginPage } from './pages/loginpage';
import { DashboardPage } from './pages/dashboardpage';

export const test = base.extend<{
    loginPage: LoginPage;
    dashboardPage: DashboardPage;
}>({
    loginPage: async ({ page }, use) => 
        await use(new LoginPage(page)),
    dashboardPage: async ({ page }, use) => 
        await use(new DashboardPage(page)),
});

// Usage in tests
test('Employee Test', async ({ loginPage, dashboardPage }) => {
    await loginPage.login('Admin', 'admin123');
    await dashboardPage.navigateToPIM();
});

Data-Driven Testing - Reading JSON Files

Data Sources

  • JSON Files - Easy to maintain
  • Excel - Business-friendly format
  • Database - Real production data
  • Parameterization - One test, many cases

From: src/data/ReadJson.ts

// Reading JSON data
import fs from 'fs';
import path from 'path';

export function readJsonData(fileName: string) {
    const filePath = path.resolve(".", fileName);
    return JSON.parse(fs.readFileSync(filePath, 'utf-8'));
}

// Data-driven tests
const users = readJsonData('data/users.json');

users.forEach((user) => {
    test(`Login: ${user.username}`, async ({ page }) => {
        await page.fill('[name="username"]', user.username);
        await page.fill('[name="password"]', user.password);
        await page.click('button[type="submit"]');
        
        const isValid = user.valid;
        await expect(page).toHaveURL(
            isValid ? /dashboard/ : /login/
        );
    });
});

API Testing with Playwright

API Testing Benefits

  • Speed - 10x faster than UI
  • Stability - No UI flakiness
  • Coverage - Test backend logic
  • CRUD - POST, GET, PUT, DELETE

From: TSSamples/PlaywrightApiTests.ts

// API Testing with Playwright
import { request } from '@playwright/test';

const apiContext = await request.newContext({
    baseURL: 'https://api.example.com',
    extraHTTPHeaders: { 'Content-Type': 'application/json' }
});

// CREATE
const postResp = await apiContext.post('/users', {
    data: { name: 'John', email: 'john@example.com' }
});
const newUser = await postResp.json();

// READ
const getResp = await apiContext.get(`/users/${newUser.id}`);

// UPDATE
await apiContext.put(`/users/${newUser.id}`, {
    data: { ...newUser, name: 'John Doe' }
});

// DELETE
await apiContext.delete(`/users/${newUser.id}`);

CI/CD Pipeline - GitHub Actions

CI/CD Benefits

  • Automation - Tests run on every push
  • Fast Feedback - Know immediately if broken
  • Parallel Jobs - Speed up execution
  • Artifacts - Save reports & videos
  • Quality Gate - Block bad code

From: .github/workflows/playwright.yml

# GitHub Actions CI/CD
name: Playwright Tests
on:
  push:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version: lts/*
    
    - run: npm ci
    - run: npx playwright install --with-deps
    - run: npx playwright test
    
    - uses: actions/upload-artifact@v4
      if: always()
      with:
        name: test-report
        path: playwright-report/

Cypress vs Playwright - When to Use What?

Cypress

  • Architecture: Runs inside browser
  • Speed: Fast feedback, live reload
  • Debugging: Time-travel, screenshots
  • API: Great cy.intercept stubbing
  • Best for: Frontend-heavy SPAs
  • Limitations: Multi-tab challenges

Playwright

  • Architecture: Controls browser externally
  • Speed: Parallel execution by default
  • Debugging: Tracing, video recording
  • API: Built-in request context
  • Best for: E2E, multi-browser, CI
  • Strengths: Native multi-tab/multi-origin

Our Choice: Both! Playwright for E2E, Cypress for rapid component testing
