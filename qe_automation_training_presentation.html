Page Object Model Pattern

Building Maintainable Test Code

POM Benefits

  • Locators in one place
  • Share across tests
  • Update once, fix everywhere
  • Readable tests
  • Easy to scale
export class LoginPage {
  #username: Locator;
  #password: Locator;
  #loginBtn: Locator;

  constructor(private page: Page) {
    this.#username = page.getByPlaceholder('Username');
    this.#password = page.getByPlaceholder('Password');
    this.#loginBtn = page.getByRole('button', { name: 'Login' });
  }

  async login(user: string, pass: string) {
    await this.#username.fill(user);
    await this.#password.fill(pass);
    await this.#loginBtn.click();
  }
}

Custom Fixtures Pattern

Dependency Injection for Clean Tests

Fixture Advantages

  • Automatic page object creation
  • No setup boilerplate
  • Parallel test isolation
  • Lazy loading
  • Easy to extend
export const test = base.extend<{
  loginPage: LoginPage;
  dashboardPage: DashboardPage;
}>({
  loginPage: async ({ page }, use) => 
    await use(new LoginPage(page)),
  dashboardPage: async ({ page }, use) => 
    await use(new DashboardPage(page))
});

// Clean test usage

Playwright vs Cypress

Choosing the Right Tool

Playwright

  • Multi-browser - Chrome, Firefox, Safari, Edge
  • True parallelism - Fast execution
  • API testing - Built-in request context
  • Mobile testing - Device emulation
  • Auto-wait - Smart element waiting
  • Multiple tabs - Easy context switching
Best for: E2E testing, cross-browser coverage, API testing

Cypress

  • Time-travel - See every command
  • Real-time reload - Instant feedback
  • Easy debugging - Developer-friendly
  • Network stubbing - Powerful cy.intercept
  • Visual runner - Interactive UI
  • Screenshots/videos - Built-in recording
Best for: Component testing, rapid development, debugging

My approach: Use both! Playwright for comprehensive E2E, Cypress for rapid component testing
